## DATABASE MANAGEMENT

### Current Tables (17 models)
Core: Device, HealthData, NetworkData, Alert
Diagnostics: WorkshopDiagnostic, DiagnosticReport
Agent: AgentHeartbeatRecord
ISP: ISPProvider, ISPStatusCheck, AgentConnectivity, ISPOutage
Automation: SystemEvent, ScheduledJob, PatchStatus, BackupStatus, NotificationLog

### Migration Strategy
- Alembic for schema changes
- deploy_migrations.sh for deployment
- Raw SQL for TimescaleDB-specific operations
- Idempotent migrations always

### Retention Policy
- Detailed data: 90 days
- Aggregated data: 2 years
- Event cleanup: monthly 1st @ 03:00

### Performance Optimization
- Connection pooling: pool_size=10, max_overflow=20, pool_pre_ping=True, pool_recycle=300
- TimescaleDB chunk_time_interval: 7 days (when enabled)
- Indexes on: device.machine_id, device.serial_number, health_data.device_id+timestamp

### Architecture Alignment Issues
1. **ORM vs asyncpg:** CLAUDE.md mandates asyncpg but code uses SQLAlchemy ORM
2. **Module structure:** CLAUDE.md mandates app/modules/{name}/ but code uses app/api/ + app/services/
3. **TimescaleDB:** May need paid tier on Render
4. **AGENT_AUTH_TOKEN:** Hardcoded in render.yaml, move to Render secret management
